# Как работать с базой знаний

## Структура

Каждый инструмент содержит 4 раздела по методологии [Diátaxis](https://diataxis.fr/):

```
инструмент/
├── index.md              # Навигация: "Я хочу..." → ссылки
├── tutorials/            # Путь обучения (читать по порядку: 01 → 02 → ...)
├── explanation/          # Понимание (читать когда хочется разобраться "почему")
├── how-to/               # Решение задач (идти сюда когда нужно сделать X)
│   └── recipes/          # Готовые конфиги (копирование → работает)
└── reference/            # Справочник (смотреть когда забыл синтаксис)
    └── cheatsheet.md     # Все команды на одной странице
```

## Типы файлов

|Тип|Вопрос читателя|Пример|
|---|---|---|
|**tutorial**|«Научи меня»|01-first-container.md|
|**explanation**|«Почему это так?»|architecture.md|
|**how-to**|«Как сделать X?»|debug-containers.md|
|**recipe**|«Дай готовый конфиг»|nginx.md|
|**reference**|«Какая команда?»|cheatsheet.md|

## Как добавить новый файл

1. Определить тип файла (tutorial / explanation / how-to / recipe / reference)
2. Скопировать шаблон из `_meta/templates/тип.md`
3. Заполнить frontmatter: `title`, `type`, `tags`, `sources`, `related`
4. Написать контент, следуя структуре шаблона
5. Добавить ссылки `[[related]]` в связанные файлы
6. Проверить по чек-листу ниже

## Чек-лист перед коммитом

### Frontmatter

- [ ] `title` — понятное название
- [ ] `type` — tutorial / explanation / how-to / recipe / reference
- [ ] `tags` — 3-8 тегов (инструмент + ключевые темы)
- [ ] `sources` — docs URL и/или книга с главой
- [ ] `related` — 2-4 ссылки на связанные файлы

### Контент

- [ ] Есть TL;DR или цель (первые строки после заголовка)
- [ ] Каждый блок кода — с комментариями
- [ ] Есть секция «Типичные ошибки» (если применимо)
- [ ] Примеры production-ready (non-root, лимиты, healthcheck)
- [ ] Нет устаревших команд
- [ ] Есть перекрёстные ссылки `[[related]]` в тексте

### Для рецептов дополнительно

- [ ] Конфиг протестирован и работает
- [ ] Есть dev и prod варианты (если применимо)
- [ ] Объяснён каждый нетривиальный выбор

## Правила обновления

- **Ошибка в теме** → править в файле, обновить `sources` если нужно
- **Новая тема** → создать файл по шаблону, добавить `related` в смежные файлы
- **Устарело** → сверить с `sources.docs` (ссылка на офиц. доку), исправить
- **Новый инсайт из книги** → добавить в соответствующую секцию, дописать `sources.book`
