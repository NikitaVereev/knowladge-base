---

При создании сервиса вы выбираете стратегию его размещения.

## 1. Replicated (Реплицированный)

Стандартный режим. Вы говорите "Хочу 3 копии", и Swarm сам решает, где их разместить.
*   **Команда:** `--replicas 3`
*   **Сценарии:** Веб-серверы, API, Worker-процессы.
*   **Поведение:** Если нода падает, Swarm перезапускает недостающие копии на других живых нодах.

## 2. Global (Глобальный)

Режим "Один на каждой ноде". Swarm запускает ровно одну копию задачи на **каждом** доступном узле кластера.
*   **Команда:** `--mode global`
*   **Сценарии:**
    *   Агенты мониторинга (Node Exporter, cAdvisor).
    *   Сборщики логов (Fluentd, Filebeat).
    *   Сетевые прокси на уровне хоста.
*   **Поведение:** При добавлении новой ноды в кластер, глобальный сервис автоматически запускается на ней.
