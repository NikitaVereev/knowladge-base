Этот туториал покажет базовый цикл работы с контейнером: запуск (run), проверка (ps), остановка (stop) и удаление (rm).

## Задача: Запустить веб-сервер Nginx

Мы запустим официальный образ Nginx в фоновом режиме.

### 1. Запуск контейнера

```bash
docker run --name my-web-server -d -p 8080:80 nginx:latest
```

**Разбор команды:**
* `docker run`: команда создания и запуска контейнера.
* `--name my-web-server`: даем контейнеру понятное имя (иначе Docker придумает случайное).
* `-d` (detach): запустить в фоновом режиме (вернуть управление терминалом).
* `-p 8080:80`: пробросить порт. Запрос на `localhost:8080` (хост) пойдет на порт `80` (контейнер).
* `nginx:latest`: имя образа и тег (версия).

### 2. Проверка работы

Проверьте, что процесс запущен:

```bash
docker ps
```
*Вы должны увидеть ваш контейнер в списке со статусом `Up`.*

Откройте в браузере или через curl:
```bash
curl http://localhost:8080
```
*Вы увидите приветственную страницу "Welcome to nginx!".*

### 3. Остановка

Остановите контейнер (процесс завершится, но настройки сохранятся):

```bash
docker stop my-web-server
```

Проверьте список *всех* контейнеров (включая остановленные):
```bash
docker ps -a
```

### 4. Удаление

Чтобы удалить контейнер окончательно (освободить имя и диск):

```bash
docker rm my-web-server
```

*Примечание: Если контейнер работает, его нельзя удалить без флага `-f` (force) или предварительной остановки.*
