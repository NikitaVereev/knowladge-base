Docker использует плагируемые драйверы для реализации сетевого взаимодействия.

## Сетевые драйверы

### Bridge (Мост) — По умолчанию
Изолирует контейнеры в частной сети, но позволяет им общаться друг с другом и с внешним миром через NAT.
* **Использование:** Стандартный выбор для большинства приложений.
* **Особенность:** Требует проброса портов (`-p`) для доступа извне.

### Host (Хост)
Убирает сетевую изоляцию. Контейнер использует сетевой стек хоста напрямую.
* **Использование:** Когда важна максимальная производительность сети или нужен доступ к диапазону портов.
* **Особенность:** Порты контейнера открываются прямо на хосте (без NAT). Если порт 80 занят на хосте, контейнер не запустится.

### None (Отсутствует)
Контейнер имеет только loopback интерфейс (`lo`).
* **Использование:** Для задач, требующих полной изоляции (безопасность, пакетная обработка данных).

### Overlay
Создает распределенную сеть между несколькими хостами Docker (Swarm или Kubernetes).
* **Использование:** В кластерах.

## Service Discovery (DNS)

В Docker встроен DNS-сервер (`127.0.0.11`), который позволяет контейнерам находить друг друга по **имени сервиса/контейнера**.

> **Важно:** DNS-резолвинг по имени контейнера работает **только в пользовательских сетях** (User-defined bridges), созданных через `docker network create`. В дефолтной сети `bridge` доступ возможен только по IP-адресам (которые могут меняться).

**Пример работы DNS:**
1. Вы создали сеть `my-net`.
2. Запустили контейнер с именем `db` в этой сети.
3. Запустили контейнер `web` в той же сети.
4. Внутри `web` приложение делает запрос к `postgres://db:5432`.
5. Docker DNS возвращает внутренний IP контейнера `db`.
