---
title: "Типы хранилищ в Docker"
description: "Разница между Named Volumes, Bind Mounts и Tmpfs. Когда и какой тип хранилища использовать."
---


Контейнеры по своей природе эфемерны: когда контейнер удаляется, данные в его Writable Layer теряются. Для сохранения данных Docker предоставляет три механизма монтирования.

## Сравнение типов

| Тип | Где хранятся данные | Управление | Персистентность | Использование |
|-----|-------------------|------------|-----------------|---------------|
| **Named Volume** | `/var/lib/docker/volumes/` | Docker | ✅ Да | Базы данных, production данные |
| **Bind Mount** | Любая папка на хосте | Пользователь | ✅ Да | Конфигурационные файлы, код в разработке |
| **Tmpfs** | Оперативная память (RAM) | Система | ❌ Нет | Секреты, временные кэши, сессии |

## Named Volumes (Рекомендуется)
Полностью управляются Docker. Вы создаете том, даете ему имя, и Docker сам решает, где его хранить.
* **Плюсы:** Легче переносить (export/import), работают на всех ОС (Linux/Windows), безопаснее (изолированы от файловой системы пользователя).
* **Минусы:** Сложнее добраться до файлов напрямую с хоста (нужны root права).

## Bind Mounts (Монтирование папок)
Прямое отображение папки с хоста в контейнер.
* **Плюсы:** Вы видите файлы прямо в своей IDE или проводнике. Изменения видны мгновенно (hot reload).
* **Минусы:** Зависит от структуры папок хоста. В продакшене может быть небезопасно, если предоставить лишний доступ.

## Tmpfs
Создает временную файловую систему в оперативной памяти.
* **Плюсы:** Очень быстро. Данные гарантированно удаляются при остановке контейнера.
* **Минусы:** Ограничено объемом RAM.
