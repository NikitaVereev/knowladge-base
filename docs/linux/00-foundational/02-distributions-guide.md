# Choosing a Distribution

## Overview

Выбор дистрибутива Linux - один из первых важных решений. Разные дистрибутивы предоставляют разные инструменты, философию и подход, но все используют один и тот же Linux kernel.

**Что вы узнаете:**
- Три главных семейства дистрибутивов (Debian, Red Hat, Arch)
- Популярные дистрибутивы в каждом семействе и их характеристики
- Разница между LTS и Rolling Release версиями
- Как выбрать дистрибутив для ваших нужд
- Команды для определения установленного дистрибутива

## Prerequisites

Перед этим разделом нужно:
- Понимание что такое Linux kernel и distribution ([[01-what-is-linux|What is Linux?]])

## Three Major Families

### 1. Debian Family: Stability and Community

**Родитель:** Debian (1993, старейший из крупных дистрибутивов)  
**Пакетный менеджер:** `apt` (Advanced Package Tool)  
**Философия:** Стабильность, свобода, поддержка сообществом

#### Популярные дистрибутивы

| Дистрибутив | Тип | Назначение |
|-------------|-----|-----------|
| **Ubuntu** | LTS + Regular | Десктоп, серверы, универсальный |
| **Debian** | Stable + Testing | Серверы, высокая стабильность |
| **Linux Mint** | LTS | Десктоп, простота для новичков |
| **Pop!_OS** | LTS | Десктоп, сфокусировано на UX |

#### Характеристики

✓ **Консервативный подход** — только проверенные технологии  
✓ **Стабильность** — приоритет над свежестью  
✓ **Огромное сообщество** — миллионы пользователей  
✓ **Много информации** — легко найти решение любой проблемы в интернете  
✓ **5-10 лет поддержки** — LTS версии получают обновления безопасности долгое время  

**Когда выбирать:**
- Новичок в Linux
- Нужна стабильность на сервере
- Хотите минимум проблем
- Нужна огромная база знаний

### 2. Red Hat Family: Innovation and Enterprise

**Родитель:** Red Hat Enterprise Linux (RHEL, платная)  
**Пакетный менеджер:** `dnf` (Dandified Yum)  
**Философия:** Инновация, корпоративная поддержка, новые технологии

#### Популярные дистрибутивы

| Дистрибутив | Тип | Назначение |
|-------------|-----|-----------|
| **Fedora** | Cutting-edge | Тестирование новых технологий |
| **Rocky Linux** | Enterprise | RHEL замена (стабильная) |
| **AlmaLinux** | Enterprise | RHEL замена (коммерческая) |

#### Характеристики

✓ **Быстрые обновления** — новые версии каждые 6 месяцев  
✓ **Новые технологии раньше** — первыми внедряют инновации  
✓ **Корпоративная поддержка** — надежны для бизнеса  
✓ **Используются на серверах** — AWS, Google Cloud и др.  
✓ **SELinux** — продвинутая система безопасности  

**Когда выбирать:**
- Нужны новые технологии
- Работаете на корпоративном сервере (RHEL-based)
- Хотите быть в тренде
- Интересуетесь enterprise Linux

### 3. Arch Family: Control and Simplicity

**Родитель:** Arch Linux (2002, минималистичный)  
**Пакетный менеджер:** `pacman`  
**Философия:** Простота (в смысле минимализма), контроль, свежесть, DIY (Do It Yourself)

#### Популярные дистрибутивы

| Дистрибутив | Стиль | Назначение |
|-------------|-------|-----------|
| **Arch Linux** | Pure | Полный контроль, опытные пользователи |
| **Manjaro** | User-friendly | Arch + удобство, рекомендуется новичкам |

#### Характеристики

✓ **Полный контроль** — вы решаете что и как установить  
✓ **Rolling release** — постоянные обновления, всегда свежие пакеты  
✓ **Минимализм** — только то что нужно, ничего лишнего  
✓ **Отличная документация** — Arch Wiki известна как лучшая в Linux  
✓ **Требует знаний** — нужно понимать как работает система  

**Когда выбирать:**
- Опытный пользователь Linux
- Хотите полный контроль над системой
- Интересует "как это работает"
- Не боитесь ломать систему и исправлять

## LTS vs Rolling Release

### LTS (Long-Term Support)

**Примеры:** Ubuntu 24.04 LTS, Debian stable

```
Версия 24.04     Версия 26.04     Версия 28.04
    |                 |                 |
    |←─ 5 лет ─→|     |←─ 5 лет ─→|     |
    |         Security Updates       |
```

**Характеристики:**
- Новая версия выпускается раз в 1-3 года
- Поддержка 5-10 лет (только security обновления)
- Очень стабильно и предсказуемо
- Используется на серверах

**Плюсы:**
- ✓ Максимальная стабильность
- ✓ Долгая поддержка
- ✓ Меньше проблем с обновлениями
- ✓ Надежно для production

**Минусы:**
- ✗ ПО может быть устаревшим
- ✗ Долгое ожидание новых версий

### Rolling Release

**Примеры:** Fedora, Arch, openSUSE Tumbleweed

```
Постоянный поток обновлений
    ↓  ↓  ↓  ↓  ↓  ↓  ↓
Каждую неделю новые пакеты
```

**Характеристики:**
- Постоянные обновления
- Всегда последнее ПО
- Нет "версий" - просто обновляется
- Может быть нестабильно

**Плюсы:**
- ✓ Всегда свежее ПО
- ✓ Новые функции сразу
- ✓ Нет "старого" железа через 3 года
- ✓ Интересно для разработчиков

**Минусы:**
- ✗ Может быть нестабильно
- ✗ Нужно часто обновляться
- ✗ Иногда что-то ломается

## How to Choose?

### По назначению

| Вы хотите | Рекомендуем |
|-----------|-----------|
| **Стабильность** | Ubuntu 24.04 LTS, Debian stable |
| **Свежесть** | Fedora, openSUSE Tumbleweed, Arch |
| **Контроль** | Arch, Gentoo |
| **Простоту** | Ubuntu, Linux Mint |
| **Сервер** | Ubuntu Server, Debian, Rocky Linux |
| **Десктоп** | Ubuntu, Fedora, Mint |
| **Производительность** | Arch, Fedora |
| **Новичок** | Ubuntu, Linux Mint, Manjaro |

## Important Commands

### Узнать установленный дистрибутив

```bash
# Полная информация о дистрибутиве
cat /etc/os-release

# Вывод будет примерно такой:
# NAME="Ubuntu"
# VERSION="24.04 LTS (Noble Numbat)"
# ID=ubuntu
# ID_LIKE=debian
```

```bash
# Альтернативный способ (на старых системах)
lsb_release -a

# Вывод:
# LSB Version:    1.4
# Distributor ID: Ubuntu
# Release:        24.04
# Codename:       noble
```

### Узнать версию Linux kernel

```bash
uname -r

# Вывод примерно:
# 6.8.0-1011-generic
```

### Информация о системе

```bash
# Красивый вывод (если установлен)
neofetch

# Альтернатива
screenfetch

# Базовая информация
uname -a
```

### На каком дистрибутиве основана ваша система?

```bash
# Посмотрите поле ID_LIKE в /etc/os-release
# debian → используйте apt, dpkg
# fedora, rhel → используйте dnf, rpm
# arch → используйте pacman
```

## Troubleshooting

### "Не знаю какой дистрибутив выбрать"

**Решение:** Начните с **Ubuntu 24.04 LTS**.

Почему:
- ✓ Самый популярный дистрибутив
- ✓ Огромное сообщество - легко найти помощь
- ✓ Хорошая документация
- ✓ Стабильный и надежный
- ✓ Подходит для новичков и профессионалов

### "Хочу попробовать несколько дистрибутивов"

**Решение:** Используйте **VirtualBox**.

Как:
1. Установите VirtualBox (бесплатно)
2. Скачайте несколько ISO образов
3. Создайте виртуальные машины для каждого дистро
4. Опробуйте и выберите понравившийся

Преимущества:
- ✓ Безопасно - не повредите основную систему
- ✓ Быстро - легко удалить и создать заново
- ✓ Реально оценить разницу

### "Своего дистрибутива нет в этом списке"

**Решение:** Определите к какому семейству он относится.

Способ:
1. Откройте `/etc/os-release` или используйте `lsb_release -a`
2. Посмотрите поле `ID_LIKE` или `PARENT_DISTRO`
3. Используйте инструкции для этого семейства

Пример:
```bash
# Zorin OS основан на Ubuntu (Debian)
# → используйте apt, инструкции для Ubuntu

# Elementary OS основан на Ubuntu (Debian)
# → используйте apt, инструкции для Ubuntu
```

### "У меня уже установлен дистрибутив, но я не знаю какой"

**Решение:**
```bash
cat /etc/os-release | grep NAME
# или
lsb_release -d
```

## Key Takeaways

- **Debian семейство** — стабильность, сообщество, для новичков
- **Red Hat семейство** — инновация, enterprise, для серверов
- **Arch семейство** — контроль, минимализм, для опытных
- **LTS** — для стабильности
- **Rolling Release** — для свежести
- **Сомневаетесь?** — выбирайте Ubuntu LTS
