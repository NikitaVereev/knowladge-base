# What is Linux?

## Overview

Linux - это бесплатная операционная система с открытым исходным кодом, которая управляет оборудованием вашего компьютера. Понимание основ Linux важно перед любыми дальнейшими действиями в этой операционной системе.

**Что вы узнаете:**
- Разница между Linux kernel и Linux distribution
- История создания и почему это важно
- Архитектура: как слои системы взаимодействуют
- Стандартная файловая система и основные директории
- Современное состояние Linux и его популярность

## Linux Kernel

**Linux kernel** — это ядро, центральная часть операционной системы, которая управляет оборудованием.

### Что делает ядро?

- **Менеджер ресурсов** — распределяет CPU, память, дисковое пространство между приложениями
- **Интерфейс** — предоставляет стандартный способ для приложений взаимодействовать с оборудованием
- **Безопасность** — обеспечивает права доступа, защищает данные пользователей друг от друга
- **Координация** — когда несколько приложений хотят использовать один ресурс, ядро координирует доступ

Ядро работает постоянно и невидимо для пользователя, управляя всеми низкоуровневыми операциями.

### Ядро как посредник

```
Application 1     Application 2     Application 3
        ↑                ↑                 ↑
        └────────────────┼─────────────────┘
                         │
                    Linux Kernel
                         │
        ┌────────────────┼─────────────────┐
        ↓                ↓                 ↓
       CPU            Memory            Disk
```

## Linux Distribution (Distro)

**Distribution** (дистрибутив) = Linux kernel + утилиты + пакетный менеджер + конфигурация + философия

### Что добавляет дистрибутив?

Ядро - это только основа. Дистрибутив добавляет:

- **Пакетный менеджер** — система для установки/обновления ПО (apt, dnf, pacman)
- **Утилиты и инструменты** — стандартные программы (ls, cp, grep, sed, awk, и т.д.)
- **Графический интерфейс** (опционально) — GNOME, KDE, Xfce или ничего
- **Философия и подход** — какие решения приняты по умолчанию

### Примеры дистрибутивов

**Ubuntu** = Linux kernel + apt + GNOME + консервативная философия (стабильность)
- Фокус на стабильность и простоту
- Выпускает новые версии каждые 6 месяцев
- LTS версии поддерживаются 5 лет

**Fedora** = Linux kernel + dnf + GNOME + инновационная философия (новые технологии)
- Фокус на свежесть и инновацию
- Тестбед для RHEL (Red Hat Enterprise Linux)
- Выпускает новые версии каждые 6 месяцев

**Arch** = Linux kernel + pacman + minimal + контроль (полный DIY)
- Фокус на простоту и контроль
- Ничего лишнего - вы собираете систему сами
- Rolling release - постоянные обновления

## История (Почему это важно)

### 1991: Начало

Молодой финский студент **Линус Торвальдс** (21 год) решил написать свою операционную систему потому что:
- UNIX стоил $5000+
- MINIX (альтернатива для обучения) имела ограничительную лицензию
- Нужна была учебная ОС для его Intel 386

### 1992: GPL - ключевое решение

Линус выпустил Linux под лицензией **GPL (GNU General Public License)**. Это было критически важно:

```
GPL лицензия означает:
✓ Код открыт - все видят что происходит
✓ Можно модифицировать - для своих нужд
✓ Можно распространять - делиться с другими
✓ Если распространяете → код остается открытым
```

Это позволило:
- Тысячам разработчиков вносить свой вклад
- Быстро находить и исправлять баги
- Создать мощное сообщество

> "Making Linux GPLed was the best thing I ever did." — Linus Torvalds

### Сегодня (2026)

- **~30 млн строк кода** в ядре
- **Тысячи разработчиков** из разных стран и компаний
- **Везде работает** — серверы, смартфоны, IoT, суперкомпьютеры
- **99% суперкомпьютеров** используют Linux

## Архитектура системы

### Слои от hardware до приложения

```
┌─────────────────────────────┐
│     Applications            │  Ваши программы, браузер, текстовый редактор
├─────────────────────────────┤
│  Desktop Environment (GUI)  │  GNOME, KDE, Xfce (опционально)
├─────────────────────────────┤
│   Shell (bash, zsh)         │  Интерпретатор команд для терминала
├─────────────────────────────┤
│ GNU Utilities               │  Стандартные утилиты: ls, cp, grep, sed, awk
├─────────────────────────────┤
│ System Libraries (glibc)    │  Интерфейс между приложениями и kernel
├─────────────────────────────┤
│   Kernel (Linux)            │  Ядро - управляет оборудованием
├─────────────────────────────┤
│   Hardware                  │  CPU, память, диск, сеть, периферия
└─────────────────────────────┘
```

Каждый слой работает через слой снизу. Приложение не может напрямую работать с CPU - оно просит kernel это сделать.

## Файловая система

### FHS (Filesystem Hierarchy Standard)

Все дистрибутивы Linux используют одну и ту же стандартную структуру файловой системы:

```
/                          корневая директория
├── /bin/                  основные команды (ls, cp, mkdir, etc)
├── /etc/                  конфигурационные файлы системы
├── /home/                 домашние папки пользователей
├── /root/                 домашняя папка администратора (root)
├── /usr/                  программы и утилиты
│   ├── /usr/bin/          пользовательские команды
│   ├── /usr/local/        локально установленное ПО
│   └── /usr/share/        общие файлы (иконки, документы)
├── /var/                  переменные данные (логи, кэши, временные файлы)
├── /tmp/                  временные файлы (очищается при перезагрузке)
├── /dev/                  файлы устройств (диски, терминалы)
├── /proc/                 информация о процессах (только в памяти)
├── /sys/                  информация о системе и драйверы
└── /boot/                 kernel и файлы загрузчика
```

Эта структура **одинакова на всех Linux системах** - это позволяет легко переноситься между дистрибутивами.

### Важные директории для пользователя

```
/home/username/        ваша домашняя папка (где ваши файлы)
/etc/                  конфиги системы (важно для администратора)
/var/log/              логи системы (когда что-то падает, смотрите сюда)
/tmp/                  временные файлы (удаляются при перезагрузке)
/opt/                  программы третьих сторон (которые вы устанавливаете)
```

## Почему Linux популярен?

### 1. Бесплатный

Нет нужно платить за лицензию. Любой может скачать, установить и использовать без ограничений.

### 2. Открытый исходный код

Весь исходный код доступен для просмотра. Вы можете:
- Видеть как это работает на самом деле
- Найти и исправить ошибки
- Модифицировать под свои нужды

### 3. Портативный

Работает на:
- Суперкомпьютерах (с петафлопс производительностью)
- Серверах (громадные data centers)
- Ноутбуках и десктопах
- Смартфонах (Android)
- Встраиваемых системах (IoT)
- Устаревшем оборудовании

### 4. Надежный

Linux системы известны своей стабильностью:
- Работают годами без перезагрузки
- Используются на критических системах (банки, больницы, аэропорты)
- Высокая uptime (время работы без сбоев)

### 5. Быстро развивается

- Постоянные обновления и улучшения
- Быстро добавляются новые возможности
- Ядро регулярно оптимизируется

### 6. Сообщество

- Тысячи разработчиков помогают добровольно
- Огромные сообщества для каждого дистрибутива
- Много документации и туториалов в интернете
- Люди помогают в форумах и чатах

## Key Takeaways

- **Linux kernel** управляет оборудованием
- **Linux distribution** добавляет утилиты, менеджер пакетов и философию
- **GPL лицензия** позволила сообществу развивать Linux
- **Архитектура слоев** позволяет приложениям не думать об оборудовании
- **Файловая система** стандартизирована на всех дистрибутивах
- **Linux везде** — серверы, облако, IoT, смартфоны
