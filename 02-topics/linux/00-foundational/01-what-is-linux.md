# Что такое Linux?

## Linux Kernel

**Linux kernel** — ядро, управляет оборудованием.

- **Менеджер ресурсов** — CPU, память, диск
- **Интерфейс** — между приложениями и железом
- **Безопасность** — права доступа, защита
- **Координация** — когда все хотят одно и то же

Ядро работает постоянно, невидимо для пользователя.

---

## Linux Distribution

**Distribution** = Kernel + утилиты + пакетный менеджер + конфигурация

Примеры:
- Ubuntu = kernel + apt + GNOME + консервативная философия
- Fedora = kernel + dnf + GNOME + инновационная философия
- Arch = kernel + pacman + ничего лишнего + полный контроль

Все используют один kernel, но добавляют разные компоненты.

## История

**1991**: Линус Торвальдс, 21 год, финский студент.
- UNIX стоит $5000+
- MINIX имеет лицензионные ограничения
- Решение: написать свою ОС

**1992**: Выпустил под GPL. Это было ключевое решение.

> "Making Linux GPLed was the best thing I ever did." — Linus Torvalds

Почему GPL важна:
- Код открыт → все видят что происходит
- Можете модифицировать
- Можете распространять
- Если распространяете → код остаётся открытым

Это позволило всем улучшать Linux вместе.

**Сегодня**: ~30 млн строк, тысячи разработчиков, везде работает.

## Компоненты системы

```
Applications
    ↑
Desktop Environment (GNOME, KDE) — опционально
    ↑
Shell (bash, zsh) — интерпретатор команд
    ↑
GNU Utilities (ls, cp, grep, sed, awk) — инструменты
    ↑
System Libraries (glibc) — интерфейс к kernel
    ↑
Kernel (Linux) — ядро
    ↑
Hardware (CPU, память, диск, сеть)
```

Каждый слой работает через слой снизу.

## Файловая система

Стандартная структура (FHS):

```
/
├── /bin          основные команды (ls, cp, etc)
├── /etc          конфигурационные файлы
├── /home         домашние папки пользователей
├── /root         домашняя папка root
├── /usr          программы и утилиты
├── /usr/bin      пользовательские команды
├── /usr/local    локально установленное ПО
├── /var          логи, кэши, переменные данные
├── /tmp          временные файлы
├── /dev          файлы устройств
├── /proc         информация о процессах
├── /sys          информация о системе
└── /boot         kernel и загрузчик
```

Эта структура одинакова на всех дистрибутивах.

## Почему Linux популярен?

1. **Бесплатный** — нет лицензионных сборов
2. **Открытый** — весь исходный код доступен
3. **Портативный** — работает везде (с разных архитектур)
4. **Надёжный** — работает годами без перезагрузки
5. **Быстро развивается** — постоянные обновления
6. **Сообщество** — люди добровольно помогают
